# 【AI×レガシーシステム保守】導入・運用ハイブリッド計画概要

## 1. はじめに（本ドキュメントの目的）
メインフレームから「OpenFrame on Linux」へのマイグレーション後において、ドキュメント類が不足している状況から、属人化を排除し、システム知識の少ない要員でも安定したシステム保守・運用を可能にするための「人とAIの共生（ハイブリッド）計画」をまとめたものです。

---

## 2. AI導入のメタ・多次元視点（4つのレンズ）
AI導入を特定の作業効率化（1次元）に留めず、保守全体の最適化を図るための4つの多次元的な設計思想です。

1. **ビジネス・戦略次元（Why）**
   * 保守コストの最適化、属人化リスクの解消、および将来にわたるシステム継続性の確保。
2. **人・組織・文化次元（People）**
   * メンバーのスキルレベルに合わせたAIの活用、AIの回答を盲信せず「ファクトチェック」する文化の醸成。
3. **プロセス・ガバナンス次元（Process）**
   * 既存の変更管理やインシデントプロセスにAIをどう組み込み、人間がどこで承認ゲート（最後の防壁）を設けるかのルール設計。
4. **技術・データ・セキュリティ次元（Technology）**
   * 移行後のソースコード、マニュアル、データ辞書の管理と、セキュリティを担保したAI環境の整備。

---

## 3. 人とAIの役割分担（棲み分け）
「人間でなければできないこと（暗黙知・判断）」と「AIが代行できること（形式知・作業）」を明確に棲み分けます。

| 領域 | 人間のタスク（対人トレーニング・OJT） | AIのタスク（RAG・アシスタント） |
| :--- | :--- | :--- |
| **環境・ルール** | ・システム開発/検証/本番環境へのアクセス申請<br>・社内セキュリティ・ガバナンスの遵守 | ・製品マニュアルや規定からの手順検索 |
| **トリアージ（初期分析）**| ・「このエラーなら、あの処理バッチを疑う」といった、長年の構造から来る**「アタリ（直感・ノウハウ）」の伝承** | ・人間が「アタリ」をつけた後の、対象PGやデータベースのコード解析・ログの日本語訳 |
| **判断・意思決定** | ・「このエラーはシステム的には軽微だが、業務締め切り直前なので最優先で対処する」といった、**ビジネス影響の優先度判断** | ・修正プログラムやデータパッチのコード生成<br>・変更による他プログラムへの影響範囲調査 |

---

## 4. 人材リソース計画と連動した、3段階のハイブリッド育成ロードマップ
移行後の要員の参画タイミングと、AIの活用レベルを同期させ、現場に無理なく導入していく段階的計画です。

### ステップ1：【初期（1〜3ヶ月目）】 人to人のOJTが主、AIはファクトチェックを通じた学習の相棒
* **対象となる人材**: 移行完了にともない、システム知識が少ない状態で新たに参画したメンバー。
* **人間の行動（OJT領域）**:
  * 環境アクセス権、セキュリティルール、基本業務フローはベテランから直接学ぶ（対人トレーニング）。
* **AIの役割**:
  * 移行後のソースコードをAIにいつでも参照・検索させ、変数名やテーブル物理名と「日本語名」の対応表（データ辞書）をAIに推測させながら人間がレビューして作成・整備する。
* **人間の成長目標（ファクトチェック能力）**:
  * AIの回答をそのまま信じず、実際のプログラムを読んで「裏取り」をする習慣をつける。これにより、メンバーは最も早く「プログラムの構文」や「システムの構造」を実体験として理解できます。

### ステップ2：【中期（4〜6ヶ月目）】 人間が「アタリ」を付け、AIが深掘りする。ドキュメントは自然に蓄積
* **対象となる人材**: 最低限の環境操作ができ、プログラムを読み解き始めた中堅メンバー。
* **人間の行動（ハイブリッド領域）**:
  * トラブル発生時、ベテランの指導のもとメンバーが「トリアージ（問題箇所の推測）」を行う。
* **AIの役割**:
  * アタリを付けた後の詳細調査（コードの解読、エラー原因の特定、影響範囲の調査）をAIが行う。
  * **ポイント**: 日々のQ&AやAIの解読結果をそのまま対応履歴（JiraやWikiなど）に保存し、これを「生きたドキュメント」として自動蓄積する（仕様書作成のための追加工数はかけない）。
* **人間の成長目標（業務フローと問いの高度化）**:
  * 局所的なコード知識を、システム全体の「業務（Why）」と結びつけて理解を深め、AIから的確な答えを引き出す「プロンプト（問い）」のスキルを磨きます。

### ステップ3：【後期（7ヶ月目〜）】 AIが一次切り分けを支援し、人間は意思決定と改善にシフト
* **対象となる人材**: システム全体を理解し、日常保守を自立して行えるようになったメンバー。
* **人間の行動（高度な判断領域）**:
  * AIの一次切り分けを参考に、人間はシステムの復旧判断や業務上の最善策の決定を行う。
* **AIの役割**:
  * ステップ2までに蓄積された「過去の対応履歴」をAIが学習。新規エラー発生時、AIが「過去の〇〇の事例に類似。対象PGはAです」と、一次的な「アタリ（トリアージ）」を提示して人間をサポートする。
* **人間の成長目標（意思決定と予防保守へのシフト）**:
  * 定型的な作業やドキュメント作成から完全に解放され、システムをより良くするための「予防保守」「機能改善」や「ビジネス上の迅速な意思決定」ができる技術者へと成長します。
